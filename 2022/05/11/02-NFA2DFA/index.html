本文借助Graphviz库实现了NFA转DFA，并输出DFA的状态转移图。


∅ Graphviz库介绍

你一定很疑惑我昨天刚介绍完Transitions库今天怎么就就换了一个绘制库。ummm…Transitions库其实更注重于状态机本身的实现，当你想用状态机来实现某些功能时，Transitions库是一个很好的选择，而绘制状态转移图只是其中的一个子功能，更重要的是无法（我不知道可不可以）显示多个接收状态（GraphMachine中的initial选项只能指定一个节点）。追根溯源，我们不难发现Transitions库其实是用Graphviz库来绘制状态转移图的，那我们为何不直接用它来绘制呢。

还是直接用一个简单直观的栗子来说明，

from graphviz import Digraph

# Graph()--Undirected Graph ; Digraph()--Directed Graph
# engine: choose layout patterns
Dot = Digraph(engine='circo')
# add nodes: you can specify color and shape of nodes
Dot.node('A')
Dot.node('B', style='filled', fillcolor='red', shape='circle')
Dot.node('C', style='filled', fillcolor='green', shape='rectangle')
# add single edge: (source,destination,label)
Dot.edge('A', 'B', 'Hello')
Dot.edge('B', 'A', 'Hola')
Dot.edge('C', 'B', 'Bye')
# add edges
Dot.edges(['AC', 'BC'])
# specify save format
Dot.format = 'png'
# save output
Dot.render('First Demo', view=True)

常见的渲染引擎有

  • Dot：默认引擎，主要用于有向图。布局算法将边缘指向相同方向（从上到下或从左到右），然后尝试避免边缘交叉并减少边长。
  • Neato：这是一种“弹簧模型”的布局，基于spring-model（namely force-based）算法，输出缺乏方向性。
  • Fdp：这是一种类似于neato的布局，是通过减少力而不是使用能量来实现的，多用于无向图。
  • Sfdp：渲染大型图，缺乏方向性。
  • Twopi：放射性布局，节点根据它们与给定根节点的距离放置在同心圆上。
  • Circo：适合绘制某些多循环结构的图表，例如某些电信网络。

有兴趣进一步了解可以参阅官方文档。

ℵ NFA 2 DFA

DFA具有确定性，无ε边，所有状态对于每一个输入字符都有唯一的跳转方向。显然，DFA的每一个状态都是若干个NFA状态的集合。

基本算法思想如下：

  1. 先求出NFA中每一个状态的（可以通过ε边到达的状态集，包括自己）并确定输入字符集（可能出现的字符）；

  2. 将开始状态的放入待更新状态集和历史状态集；

  3. 从待更新状态集里取出一个状态，计算读入每一个字符（）时跳转到的目的状态

    其中表示NFA中状态读入字符后跳转到的状态；

  4. 若目的状态未出现在历史状态中，那么将其加入待更新状态列表，同时加入历史状态集。每次都从待更新状态集中取出一个状态重复上述步骤，直至待更新状态集为空。

可以参阅这一个例子：Converting an NFA to a DFA - Example

C☺DE

from graphviz import Digraph

num = eval(input('请输入状态转移函数的个数：'))
start = input('请输入初始状态序号：')
# end is a list
end = set(input('请输入所有接收状态序号：').split())
symbols = set()
# transfer[sym][source]=dest ⇔ (source)--sym-->(dest)
transfer = dict()
closure = dict()
print('请依次输入【trigger】【source】【destination】:')
for i in range(num):
    a, b, c = input().split()
    closure.setdefault(b, {b})
    closure.setdefault(c, {c})  # 防止终结节点没有入度
    # '#' is equivalent with 'ε'
    if a != '#':
        symbols.add(a)
        transfer.setdefault(a, dict()).setdefault(b, {c}).add(c)
    else:
        closure[b].add(c)

# 上述代码得到的closure只是邻接ε节点，还需要迭代计算真正的ε-closure
for state in closure:
    upd = closure[state]
    # upd是每一轮新增节点的集合
    # 当upd不为空时需要继续迭代更新
    while upd:
        closure[state] |= upd
        nxt = set()
        for n in upd:
            nxt |= closure[n]
        upd = nxt - closure[state]

# 历史状态集
DFA_states = [closure[start]]
# 待更新状态集
to_update = [closure[start]]
transitions = []
while to_update:
    status = to_update.pop()
    # status的状态转移列表
    single_trans = []
    for sym in symbols:
        new = set()
        tmp = set()
        for s in status:
            tmp |= transfer[sym].setdefault(s, set())
        for s in tmp:
            new |= closure[s]
        if new not in DFA_states:
            to_update.append(new)
            DFA_states.append(new)
        # 将具有相同source和dest的有向边label进行合并
        flag = True
        for t in single_trans:
            if t[2] == str(new):
                t[0] = t[0] + '|' + sym
                flag = False
                break
        if flag:
            single_trans.append([sym, str(status), str(new)])

    transitions.extend(single_trans)

DFA = Digraph(engine='dot')
DFA.edge('start', str(DFA_states[0]))
for n in DFA_states:
    if n & end:
        DFA.node(str(n), style='filled', fillcolor='green', shape='rectangle')
    else:
        DFA.node(str(n), shape='rectangle')
for t in transitions:
    DFA.edge(t[1], t[2], label=t[0])
DFA.format = 'png'
DFA.render('DFA', view=True)
