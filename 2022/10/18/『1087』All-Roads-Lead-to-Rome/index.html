『1087』All Roads Lead to Rome

2022/10/18 PAT（Advanced Level）Practice C＋＋ Graph Dijkstra

Indeed there are many different tourist routes from our city to Rome. You are supposed to find your clients the route with the least cost while gaining the most happiness.

Input Specification:

Each input file contains one test case. For each case, the first line contains 2 positive integers N (2≤N≤200), the number of cities, and K, the total number of routes between pairs of cities; followed by the name of the starting city. The next N−1 lines each gives the name of a city and an integer that represents the happiness one can gain from that city, except the starting city. Then K lines follow, each describes a route between two cities in the format City1 City2 Cost. Here the name of a city is a string of 3 capital English letters, and the destination is always ROM which represents Rome.

Output Specification:

For each test case, we are supposed to find the route with the least cost. If such a route is not unique, the one with the maximum happiness will be recommanded. If such a route is still not unique, then we output the one with the maximum average happiness – it is guaranteed by the judge that such a solution exists and is unique.

Hence in the first line of output, you must print 4 numbers: the number of different routes with the least cost, the cost, the happiness, and the average happiness (take the integer part only) of the recommanded route. Then in the next line, you are supposed to print the route in the format City1->City2->...->ROM.

Sample Input:

6 7 HZH
ROM 100
PKN 40
GDN 55
PRS 95
BLN 80
ROM GDN 1
BLN ROM 1
HZH PKN 1
PRS ROM 2
BLN HZH 2
PKN GDN 1
HZH PRS 1

Sample Output:

3 3 195 97
HZH->PRS->ROM

好久没Dijkstra了，这道题要输出的东西有亿点多。

从一个城市去罗马，每条路上有一定的代价，到一座城市能收获相应的快乐值，输出【代价最短的路线数 最小代价 最小代价中的最大快乐值 平均快乐值】和最优路线（评价优先级依次为：代价最小，快乐多多，城市少少）。

首先用Dijkstra求出所有代价最小的路线，老套路，最短路线上的每个节点存储上一个节点的编号，由于路线不唯一，所以每个节点可能有多个前节点，用vector进行存储。然后从终点DFS回溯，回溯的路上顺便计算一下快乐值和节点个数，然后记录每个节点的最优next节点，即如果回到某个城市得到了一个更优的，那么就更新其next值，回溯到起点0就停止。那么最后最优的快乐值和节点数就存储在起点0的info中。

第一次提交case 3超时，看到网上也没有人case 3裂开的，于是乎我改进了回溯函数，对于已经回溯过的节点不再重新回溯，直接用之前得出的结果，但绿色依旧，直到我把所有的INT16_MAX换成了INT32_MAX。INT_MAX是用来代替Dijkstra中走不通距离的，有时候替代数太小了会出问题。

⚠️ 尽量用大数（INT32_MAX）替代。

🐎

#include <iostream>
#include <map>
#include <vector>

using namespace std;

vector<int> happy, nxt, path;
vector<vector<int>> pre;     // previous nodes in the least cost paths
vector<pair<int, int>> info; // <happiness sum, nodes num>

int collect(int &node)
{
    int sum = 0;
    for (auto k: pre[node])
    {
        auto p = make_pair(info[node].first + happy[k], info[node].second - 1);
        if (p > info[k])
        {
            info[k] = p;
            nxt[k] = node;
        }
        sum += (path[k] == -1 ? collect(k) : path[k]);
    }
    return path[node]=sum;
}

int main()
{
    int n, k, dst, c;
    scanf("%d %d", &n, &k);
    vector<string> city(n, string(3, 0));
    happy.resize(n, 0);
    map<string, int> num;
    cin >> city[0];
    num[city[0]] = 0;
    for (int i = 1; i < n; ++i)
    {
        scanf("%s %d", &city[i][0], &happy[i]);
        num[city[i]] = i;
        if (city[i] == "ROM")
            dst = i;
    }
    vector<vector<pair<int, int> > > cost(n);
    string c1(3, 0), c2(3, 0);
    for (int i = 0; i < k; ++i)
    {
        scanf("%s %s %d", &c1[0], &c2[0], &c);
        cost[num[c1]].emplace_back(num[c2], c);
        cost[num[c2]].emplace_back(num[c1], c);
    }
    vector<int> dist(n, INT32_MAX);
    vector<bool> done(n, false);
    dist[0] = 0, done[0] = true;
    pre.resize(n);
    for (auto &v: cost[0])
        pre[v.first].push_back(0), dist[v.first] = v.second;
    while (!done[dst])
    {
        int min_d = INT32_MAX;
        vector<int> opt;
        for (int i = 1; i < n; ++i)
        {
            if (done[i]) continue;
            else if (dist[i] < min_d)
            {
                opt.clear();
                opt.push_back(i);
                min_d = dist[i];
            }
            else if (dist[i] == min_d)
                opt.push_back(i);
        }
        for (auto &o: opt)
        {
            for (auto &v: cost[o])
                if (dist[o] + v.second < dist[v.first])
                {
                    dist[v.first] = dist[o] + v.second;
                    pre[v.first].clear();
                    pre[v.first].push_back(o);
                }
                else if (dist[o] + v.second == dist[v.first])
                    pre[v.first].push_back(o);
            done[o] = true;
        }
    }
    info.resize(n, {INT32_MIN, INT32_MIN});
    info[dst] = make_pair(happy[dst], 0);
    nxt = path = vector<int>(n, -1);
    path[0] = 1;
    printf("%d %d %d %d\n", collect(dst), dist[dst], info[0].first, info[0].first / -info[0].second);
    int idx = 0;
    printf("%s", city[0].c_str());
    while ((idx = nxt[idx]) != -1)
        printf("->%s", city[idx].c_str());
}

Tips

  1. 注意到回溯函数中节点数我是-1处理的，因为快乐数要bigger，而节点数要尽量smaller，如果要使用pair的比较运算就需要统一两者，所以对节点数取了个负

  2. 对所有城市先进行了编号，map用来获取编号，vector存取编号对应城市名

  3. 距离用了邻接表的结构进行存储，Dijkstra用邻接表足矣
